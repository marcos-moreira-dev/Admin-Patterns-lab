# QA, testing y validación

## QA
QA significa aseguramiento de calidad. No es solo testing, sino la disciplina de revisar si el software cumple con calidad razonable.

## Qué entra aquí
- pruebas funcionales
- pruebas de flujos principales
- validación de requisitos
- revisión de casos raros
- regresión
- control de cambios

## Testing
Es una de las herramientas principales de QA.

### Tipos útiles para este contexto
- pruebas unitarias
- pruebas de integración
- pruebas de casos de uso
- pruebas manuales guiadas por flujo
- pruebas de aceptación con el cliente

## En sistemas administrativos pequeños y medianos
No necesitas un ejército de QA, pero sí una disciplina básica:
- probar happy path
- probar edge cases
- revisar permisos
- revisar mensajes de error
- verificar que no rompiste algo que antes funcionaba
