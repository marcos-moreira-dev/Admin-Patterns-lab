# Flujo de trabajo con IA para proyectos reales

Este bloque propone usar la IA como parte del proceso, no como reemplazo del proceso.

## Flujo recomendado

1. Levantamiento y notas propias.
2. Limpieza del material relevante.
3. Biblioteca o repositorio de referencia.
4. Borrador con IA local o cloud según el caso.
5. Revisión humana.
6. Conversión a documento, código o propuesta.
7. Retroalimentación del cliente o del proyecto.
8. Incorporación a la biblioteca para no perder aprendizaje.

## Dónde aporta más

- estructurar Markdown
- generar esqueletos de código
- revisar coherencia
- resumir reuniones
- producir variantes de texto
- proponer listas de chequeo

## Dónde nunca debería mandar sola

- alcance contractual
- cobros
- seguridad crítica
- decisiones de datos irreversibles
- cambios legales o tributarios
- migraciones delicadas en producción
