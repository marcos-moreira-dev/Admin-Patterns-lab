# Cómo leer este bloque

Este bloque parte de una idea simple: cuando un cliente ya usa software, empiezan a aparecer oportunidades reales de mejora. La tarea profesional no es vender por vender, sino detectar dolor operativo, justificar valor y proponer fases con criterio.

## Orden recomendado
1. `01_por_que_el_software_crece_con_el_negocio.md`
2. `02_como_detectar_oportunidades_reales_de_mejora.md`
3. `03_como_presentar_mejoras_sin_sonar_vendedor.md`
4. `04_como_justificar_el_valor_de_una_nueva_fase.md`
5. `07_como_cobrar_evoluciones_sin_confundirlas_con_mantenimiento.md`
6. `10_checklist_para_proponer_una_mejora.md`

## Documento madre
