# Propósito y alcance de este bloque

Este bloque no habla de mantenimiento mensual, sino de evolución con valor.

## Tesis principal

**las mejoras de software deben proponerse como respuesta a problemas operativos observados, no como capricho técnico ni como venta vacía.**

## Qué intenta dejar claro

- por qué un sistema crece con el negocio
- cómo detectar oportunidades reales de mejora
- cómo justificar una nueva fase
- cómo distinguir mantenimiento de evolución
- cómo presentar mejoras sin sonar vendedor de humo
