# Checklist de salida a producción

## Antes de desplegar
- [ ] alcance de la versión claro
- [ ] build exitoso
- [ ] pruebas mínimas ejecutadas
- [ ] migraciones revisadas
- [ ] variables de entorno definidas
- [ ] secretos fuera del código
- [ ] versión identificable
- [ ] backup disponible si aplica

## Durante el despliegue
- [ ] artefacto correcto
- [ ] entorno correcto
- [ ] conexión a base validada
- [ ] logs iniciales revisados
- [ ] healthcheck o validación básica ejecutada

## Después del despliegue
- [ ] login funciona
- [ ] flujo principal funciona
- [ ] módulos críticos responden
- [ ] errores básicos visibles y entendibles
- [ ] versión documentada
- [ ] cliente o responsable informado

## Regla final
Producción no debería ser una aventura improvisada.
