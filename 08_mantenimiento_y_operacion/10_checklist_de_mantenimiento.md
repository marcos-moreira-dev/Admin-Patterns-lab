# Checklist de mantenimiento

## Antes de tocar algo
- ¿entiendo si es bug, ajuste o evolución?
- ¿puedo reproducirlo o delimitarlo?
- ¿hay evidencia mínima?
- ¿afecta datos críticos?
- ¿afecta seguridad o permisos?
- ¿afecta contratos API o estados?

## Antes de desplegar corrección
- ¿validé el flujo afectado?
- ¿revisé si rompe algo existente?
- ¿requiere migración?
- ¿dejé trazabilidad del cambio?
- ¿el cliente sabe qué se corrigió y qué no?

## Después de corregir
- ¿el incidente quedó documentado?
- ¿quedó claro el impacto?
- ¿apareció deuda técnica asociada?
