# Metodología de trabajo con clientes

## Propósito

Este bloque explica cómo conducir una relación profesional con un cliente sin improvisar, sin sonar vendedor agresivo y sin saltar demasiado pronto a pantallas o arquitectura. La idea es mantener una conversación útil, ordenada y progresiva.

## Tesis de trabajo

La relación con el cliente se sostiene mejor cuando cada conversación tiene un objetivo concreto, una salida verificable y un siguiente paso claro.

No se conversa para impresionar. Se conversa para:
- entender el negocio
- detectar dolor real
- ordenar alcance
- validar prioridades
- construir confianza

## Principios de conversación

### 1. Primero entender, luego proponer
Antes de hablar de módulos, colores o pantallas, conviene entender operación, flujo, restricciones y dolores.

### 2. No vender complejidad
El cliente debe sentir que se le está ayudando a ordenar un problema, no a comprar un sistema inflado.

### 3. Guiar, no interrogar
La conversación no debe parecer examen. Hay que pedir ejemplos, narraciones y casos reales.

### 4. Siempre cerrar con claridad
Cada reunión debe terminar con:
- qué se entendió
- qué falta validar
- qué se hará después

## Mapa general de la relación

1. **Primer contacto**: abrir confianza y detectar si hay problema real.
2. **Descubrimiento**: entender negocio, dolor y valor esperado.
3. **Levantamiento**: bajar a dominio, flujos, datos, reglas y prioridades.
4. **Devolución**: resumir lo entendido y validar.
5. **Propuesta**: recortar alcance, definir MVP o V1 y siguiente paso.
6. **Evolución**: proponer mejoras posteriores sobre uso real.
