# Dominio doméstico trivial para el laboratorio

Para no distraer el análisis con un negocio complejo, el laboratorio usa un dominio doméstico trivial. La idea es tener datos fáciles de entender, pero suficientemente variados para ilustrar tipos de módulo.

## Entidades ligeras sugeridas

- tareas del hogar
- compras domésticas
- pagos de servicios
- incidencias de la casa
- documentos del hogar
- agenda de mantenimiento

## Ejemplos por tipo de módulo

### Dashboard
- gastos del mes
- tareas pendientes
- cosas urgentes en casa

### CRUD
- productos de limpieza
- categorías de compra
- habitaciones o espacios

### Wizard
- registrar compra grande
- registrar mantenimiento de un electrodoméstico

### Expediente
- seguimiento de una fuga de agua
- seguimiento de una reparación doméstica

### Bandeja
- pendientes por atender
- tareas vencidas o en espera

### Documental
- facturas del hogar
- garantías
- manuales

### Agenda
- pagos próximos
- recordatorios de mantenimiento

## Criterio de uso

El dominio doméstico es solo soporte didáctico. Los nombres deben seguir siendo genéricos y reutilizables. El foco está en el patrón, no en el negocio.
