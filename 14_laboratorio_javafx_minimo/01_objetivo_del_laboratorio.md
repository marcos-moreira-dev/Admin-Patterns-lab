# Objetivo del laboratorio JavaFX mínimo

Este bloque define un laboratorio visual y técnico para practicar patrones administrativos en JavaFX sin dominio de negocio pesado y sin base de datos.

## Propósito

El objetivo no es construir un sistema real, sino una plantilla demostrativa donde cada tipo de módulo administrativo tenga:

- una categoría clara
- una o más variantes razonables
- nombres de componentes autoexplicativos
- datos en memoria
- layout sobrio y reutilizable
- una estructura que luego pueda copiarse o adaptarse a otros proyectos

## Lo que sí busca este laboratorio

- fijar visualmente los patrones administrativos
- servir como base para copiar y adaptar
- permitir experimentar con CRUD, wizard, expediente, bandeja y otros módulos
- practicar navegación, shell, topbar, sidebar y workspace central
- mostrar cómo aterrizar patrones en FXML, controllers y datos en memoria

## Lo que no busca

- resolver un dominio real profundo
- incluir seguridad real, base de datos o backend real
- cubrir CSS avanzado o diseño gráfico sofisticado
- convertirse en framework rígido

## Resultado esperado

Una aplicación JavaFX mínima, navegable y entendible, donde el desarrollador pueda abrir una categoría, ver variantes y tomar piezas como plantilla para proyectos posteriores.
