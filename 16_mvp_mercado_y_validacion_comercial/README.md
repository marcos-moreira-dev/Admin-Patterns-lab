# Cómo leer este bloque

Este bloque no trata de programar más rápido por sí mismo. Trata de decidir mejor qué construir primero, qué cortar, qué ofrecer, qué cobrar y cuándo dejar de pensar como prototipo para empezar a pensar como producto usable.

## Orden recomendado
1. `00_proposito_y_alcance.md`
2. `01_que_es_un_mvp_en_software_administrativo.md`
3. `02_que_si_meter_y_que_no_meter_en_un_mvp.md`
4. `05_que_minimo_debo_ofrecer_para_cobrar_sin_miedo.md`
5. `06_como_explicarle_al_cliente_que_el_software_es_iterativo.md`
6. `11_como_cobrar_un_mvp_o_v1_sin_regalar_el_trabajo.md`

## Documento madre
El documento madre de este bloque es `01_que_es_un_mvp_en_software_administrativo.md`. Los demás lo amplían desde ángulos de nicho, validación, alcance, cobro y evolución.

## Advertencia editorial
Este bloque no defiende construir software pobre. Defiende construir lo suficiente para validar, cobrar y aprender sin hipotecar meses en complejidad ornamental.
