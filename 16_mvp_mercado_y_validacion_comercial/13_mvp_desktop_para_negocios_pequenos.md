# MVP desktop para negocios pequeños

## Cuándo tiene sentido

Una app desktop administrativa tiene mucho sentido cuando:

- el negocio opera desde un punto fijo
- el usuario principal es administrativo
- la formalidad visual importa
- no se necesita movilidad extrema
- el entorno de uso es predecible

## Qué debe priorizar un MVP desktop

- claridad visual
- velocidad operativa
- flujo principal completo
- pocos pasos
- navegación estable
- sensación de formalidad

## Qué no conviene complicar al inicio

- efectos visuales innecesarios
- arquitectura de UI demasiado elaborada
- estados frontend sofisticados sin necesidad
- componentes genéricos por ansiedad

## Qué sí conviene cuidar

- layout claro
- tablas legibles
- formularios correctos
- mensajes comprensibles
- botón y acción bien identificados
- búsqueda mínima

## Regla práctica

Un MVP desktop puede ser sobrio y aún así verse profesional. Lo importante es que inspire confianza y ayude a operar.
