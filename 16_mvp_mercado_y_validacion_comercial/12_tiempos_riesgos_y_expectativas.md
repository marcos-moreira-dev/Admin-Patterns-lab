# Tiempos, riesgos y expectativas

## Tiempos

Un MVP serio no se promete como si fuera instantáneo, pero tampoco se infla como si fuera una empresa enorme. Los tiempos deben presentarse como estimación razonable, no como juramento ciego.

## Riesgos que debes mencionar

- falta de claridad del cliente
- cambios de alcance
- dependencia de decisiones pendientes
- datos incompletos
- complejidad oculta descubierta durante el proceso
- ajustes por uso real

## Cómo hablar de tiempos sin vender humo

> Este tiempo supone que el alcance actual se mantiene y que las decisiones clave se responden a tiempo. Si el problema cambia de tamaño, la planificación también cambia.

## Expectativas correctas

El cliente debe esperar:
- una primera versión útil
- no una solución total de una vez
- posibilidad de evolución
- necesidad de ajustes con uso real

## Lo que tú debes esperar

- cambios de percepción del cliente al ver algo funcionando
- redefinición de prioridades
- aparición de mejoras obvias que antes no se veían
