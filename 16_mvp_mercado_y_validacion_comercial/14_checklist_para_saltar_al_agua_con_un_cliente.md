# Checklist para saltar al agua con un cliente

## Antes de proponer

- ¿entiendo el dolor principal?
- ¿puedo explicar el flujo central?
- ¿sé qué no entra?
- ¿sé qué sí se puede cobrar con dignidad?

## Antes de construir

- ¿el alcance está recortado?
- ¿hay un flujo principal claro?
- ¿sé qué actor usará primero el sistema?
- ¿sé qué datos mínimos necesito?

## Antes de cobrar

- ¿puedo explicar el valor sin esconderme detrás de la palabra software?
- ¿puedo justificar por qué esta primera fase ya sirve?
- ¿dejé claro qué no está incluido?

## Antes de entregar

- ¿el flujo principal funciona de inicio a fin?
- ¿el sistema se puede usar sin estar yo al lado todo el tiempo?
- ¿hay una base para la siguiente fase?

## Señal de que sí puedes avanzar

Si puedes explicar con claridad:
- problema
- flujo
- alcance
- valor
- recorte
- siguiente paso

entonces ya puedes entrar con más seguridad.
